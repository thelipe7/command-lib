# Contributing to `command-lib`

Thanks for considering a contribution.

## Ways to Contribute

- report bugs with the issue templates
- request features with the issue templates
- ask usage questions with the `Question` template
- open pull requests for fixes, documentation, or improvements

## Before You Start

- check the [documentation](https://github.com/thelipe7/command-lib/wiki) for existing guidance
- search existing issues and pull requests before opening a new one
- keep changes focused on one problem or improvement at a time

## Development Notes

- target Java 21
- keep compatibility aligned with the Paper version documented in the repository
- preserve the current public API unless the change is intentional and documented
- update documentation when behavior, setup, or examples change

## Pull Requests

- describe the problem and the change clearly
- link related issues when applicable
- include usage examples when the change affects public behavior
- update the relevant documentation page when public behavior changes
- keep formatting and naming consistent with the existing codebase

## Documentation Changes

If you change:

- command registration behavior
- argument resolution behavior
- tab completion behavior
- permission or message behavior
- public examples or setup steps

update the matching page in the documentation together with the code change.

## Support Scope

Use issues for:

- reproducible bugs
- concrete feature requests
- usage questions related to the library

Avoid opening issues for unrelated plugin code or general Java support questions that are outside this project.
